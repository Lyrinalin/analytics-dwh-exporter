# Analytics DWH Exporter (TypeScript)

Скрипт (`analyticsJob.ts`) работает как мини-ETL (Extract, Transform, Load) пайплайн: он скачивает "заказы" из внешнего API, отфильтровывает невалидные данные, агрегирует финансовые метрики и сохраняет итоговый дамп в JSON (имитация подготовки данных для DWH, например, Google BigQuery).

## Что делает скрипт?

1. **Extract (Извлечение):**
   - Выполняет HTTP GET запрос к публичному мок-API через нативный `fetch`.
   - В случае недоступности внешнего API (например, если мок удален), бесшовно переключается на фолбек-данные (`generateFallbackMockData`), чтобы пайплайн отработал без сбоев.

2. **Transform (Трансформация):**
   - Отбрасывает отмененные заказы (`status === 'CANCELED'`).
   - Фильтрует заказы по строго заданной дате — оставляет только покупки за **предыдущий календарный месяц** относительно текущей даты выполнения скрипта.
   - Считает общую валовую выручку (Total Revenue) по оставшимся валидным заказам.
   - Рассчитывает комиссию маркетплейса (задается в константах, по умолчанию `15%`).
   - Группирует объем проданных позиций (quantity) по категориям товаров.

3. **Load (Загрузка):**
   - Асинхронно сохраняет итоговую агрегированную аналитику в файл `analytics_export_bigquery.json`.
   - Выводит красивую сводку и время выполнения скрипта в консоль.

## Технологии и подходы (Как это реализовано)

- **TypeScript (Strict Mode):** Использование `interface`, `type` (например, Union: `'COMPLETED' | 'CANCELED'`), явного указания типов возвращаемых функций. Никаких `any`.
- **Data Casting:** Безопасное приведение ответов от API к нужным интерфейсам (например, `await response.json() as Order[]`).
- **Современный асинхронный Node.js:**
  - `async/await` конструкции.
  - Нативный `fetch` (Node 18+).
  - Модуль `fs/promises` для безопасной и неблокирующей работы с файловой системой.
- **Оптимизация производительности:** Использование цикла `for...of` для одной итерации по большому массиву вместо ресурсоемких цепочек `.filter().map().reduce()`.
- **Профилирование:** Замер производительности скрипта через `performance.now()`.

## Как пользоваться

### 1. Требования

Для запуска потребуется установленный на компьютере **Node.js** (желательно версии 18 и выше).

### 2. Установка зависимостей

Откройте терминал в папке проекта (где лежит файл `analyticsJob.ts`) и установите нужные типы и инструменты:

```bash
npm install
```

*(Если вы еще не инициализировали проект, выполните `npm init -y; npm i --save-dev @types/node tsx typescript`)*

### 3. Запуск

Скрипт можно запустить без предварительной компиляции (превращения `.ts` в `.js`) с помощью утилиты `tsx`, которая была установлена на предыдущем шаге:

```bash
npx tsx analyticsJob.ts
```
